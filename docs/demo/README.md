# Demo: README-driven mini workflow

このサンプルはテンプレートの使い方を実演するリポジトリ内のワークフローを記録したドキュメントです。`docs/demo/` 以下には、`samples/demo/` にある完成例を段階ごとに切り出した内容を収めています。

## フローの概要

1. `PLAN.md` で目的とアウトプット、背景を言語化する。
2. `SPEC.md` で作業の進め方・スコープ・ステップを整備する。
3. `TODO.md` でタスクを整理し、優先度や状態を付与する。
4. `KNOWLEDGEMENT.md` で参照すべき知識や資料、前提を確認する。
5. `CHECKPOINT.md` にセッションの記録と次のアクションを残す。

## このドキュメントが示すこと

- 各テンプレートを埋めていく順番がわかるという観点で README を主軸に考えた解説。
- docs ディレクトリ内で実際のプロジェクトに敷衍する前に、`samples/demo` にある記入済み例を参照しながら各セクションを整える。
- セッションごとに `CHECKPOINT.md` を更新し、「何をしたか」「何をするか」「意図」を残す習慣を体得する。

## サンプル運用のヒント

- この README をコピーして `samples/<your-project>/README.md` から書き始めると、手順を追いやすい。
- 各ドキュメントは `docs/demo/` 以下で独立した単位（PLAN、SPEC、TODO、KNOWLEDGEMENT、CHECKPOINT）として記録することで、後で部分的に参照しやすくなる。
- `samples/demo/` の内容を参考にしつつ、テンプレートのコメントに従って自分のプロジェクトに合わせて順番に埋めていく。今書いているこの README はそのガイド例です。
